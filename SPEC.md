# 레터링 — 사진 위 텍스트 편집 웹앱 (SPEC)

작성일: 2026-07-19

## 개요
사진을 불러와 그 위에 손글씨/레터링 폰트로 텍스트를 얹고, 원본 해상도로 다운로드하는 정적 웹앱.

## 스택 / 배포
- **정적 웹앱**: Vanilla JS (ES 모듈) + HTML/CSS + Canvas. 빌드 도구 없음(no-build).
- **타겟 브라우저**: Chrome / Edge (다운로드에 File System Access API 사용, 미지원 시 `<a download>` 폴백).
- **배포**: GitHub Pages (무료). 서버/백엔드 없음 → 고정비 0.
- **처리**: 100% 클라이언트. 사진은 업로드되지 않고 브라우저 내에서만 처리.

## 화면 레이아웃 (2분할)
- 상단 툴바: 왼쪽 = 로고 이미지(추후 첨부, 지금 placeholder), 오른쪽 = **다운로드** 버튼.
- 왼쪽: 사진 + 편집 캔버스 영역.
  - **빈 상태(empty state)**: "사진 불러오기" 드롭존(클릭 + 드래그앤드롭). 사진이 없을 때만 노출.
  - 사진 로드 시 같은 자리가 편집 캔버스(stage)로 전환. 불러오기 UI는 사라짐.
- 오른쪽 패널: 상단 라벨 **"글자"**(폰트 리스트) + 하단 **컬러**.

## 폰트 (번들, 큐레이션, OFL)
Gaegu(개구) / Nanum Pen Script(나눔펜) / Gamja Flower(감자꽃) / Nanum Myeongjo(나눔명조) /
Noto Serif KR(노토명조, variable→wght 500 인스턴싱) / Gowun Dodum(고운돋움) — 한글 6종.
- 각 리스트 항목은 날짜 플레이스홀더 `2026.07.19`를 해당 폰트로 미리보기.
- 항목 클릭 → 사진 위에 그 폰트의 텍스트 박스 생성.

## 컬러
- 화이트 / 블랙 / RGB 직접 선택(color picker). (배경 기반 추천 팔레트는 MVP 제외)
- 현재 **선택된 텍스트 박스**에 적용.

## 텍스트 박스 상호작용
- **이동**: 박스 몸통 드래그.
- **삭제**: 좌상단 X 버튼.
- **회전**: 우상단 버튼 드래그(중심 기준).
- **크기**: 우하단 버튼 드래그(폰트 크기 조절).
- **편집**: 박스 내부 클릭 → 커서 깜빡, 텍스트 입력(플레이스홀더 `2026.07.19`).
- **완료/해제**: 다른 영역 클릭 → 박스 테두리·버튼 숨김, 글자만 사진 위에 남음.
- **재편집**: 남은 글자 다시 클릭 → 박스 재선택.
- 여러 개 박스 추가 가능. 한 번에 하나 선택.

## 좌표계 & 저장
- stage = 화면에 표시되는 이미지 크기(종횡비 유지해 영역에 맞춤). 텍스트 박스 좌표는 stage 픽셀 기준(center cx/cy, fontSize, rotation).
- **다운로드**: 원본 해상도(naturalWidth×naturalHeight) 캔버스에 원본 이미지 + 텍스트를 `ratio = naturalWidth/stageWidth` 배율로 다시 렌더 → PNG(기본)/JPG. 화질 손실 없음.

## 에러 처리
- 이미지가 아닌 파일 → 안내 메시지, 무시.
- 텍스트 박스 없이 다운로드 → 원본 이미지만 저장(정상).
- 폰트 로드 완료(`document.fonts.ready`) 후 미리보기/저장 렌더.

## 검증 (성공 기준)
1. 사진 불러오기(클릭/드롭) 동작.
2. 폰트 선택 → 박스 생성 → 이동/회전/크기/삭제 동작.
3. 텍스트 입력 → 다른 곳 클릭 시 글자만 남고, 재클릭 시 재편집.
4. 컬러(화이트/블랙/RGB) 적용.
5. 다운로드 결과가 화면과 시각적으로 일치, 원본 해상도.
6. Windows Chrome/Edge에서 최종 확인(사용자 환경).
